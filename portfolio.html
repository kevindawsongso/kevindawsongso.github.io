Vulnerability Management & Compliance Framework Lab

Project Snapshot
Simulated real-world vulnerability testing to evaluate system compliance against industry-standard security frameworks like NIST, ISO 27001, and OWASP Top 10.

Context & Motivation
The project aimed to explore how vulnerability scanning aligns with governance frameworks and control baselines, bridging technical execution with risk and compliance strategies.

Thought Process & Decision-Making
Systems with known weaknesses were scanned using Nessus and OpenVAS, focusing on detecting gaps against predefined controls. Findings were mapped to NIST and ISO controls to reveal inconsistencies in patching cadence and access controls. Fixes were prioritized based on exploitability and regulatory relevance.

Technical Highlights

  • Used Tenable Nessus to detect critical vulnerabilities across Windows and Linux hosts.
  • Correlated findings with OWASP Top 10 and CIS benchmarks.
  • Created a remediation plan and measured improvements by re-running scans.
  • Logged all vulnerabilities and mitigations for audit reporting.

What I Learned / What’s Next
The lab deepened the understanding of how technical risk maps to compliance obligations. Future plans include automating baseline checks using scripts and integrating scan results with SIEM alerts.

AWS Cloud Security & Threat Detection Lab

Project Snapshot
Designed and deployed a cloud honeypot on AWS to collect real-world attack data and refine detection strategies using native AWS tools.

Context & Motivation
The project provided hands-on experience with cloud-native security controls and real-world telemetry, offering insight into attacker behavior and AWS threat visibility.

Thought Process & Decision-Making
A T-Pot honeypot was deployed on an EC2 instance to simulate vulnerable services, with traffic segmented using VPCs and security groups. AWS GuardDuty and Detective were used for detection and correlation, and AWS WAF rules were tuned in response to observed brute-force attempts.

Technical Highlights

  • Deployed T-Pot honeypot stack on EC2.
  • Configured VPC Flow Logs, GuardDuty, and Detective for threat telemetry.
  • Captured and analyzed RDP/SSH brute-force attempts.

What I Learned / What’s Next
The project demonstrated how AWS services interconnect for layered cloud defense. Future plans involve exploring Lambda automation for response and using Athena to query logs for long-term patterns.

SIEM & IDS/IPS Cybersecurity Simulation Lab

Project Snapshot
Built a hybrid SOC-style environment to monitor real-time security events, simulate threats, and execute incident response.

Context & Motivation
The project aimed to get closer to real-world SOC operations by practicing event triage, log correlation, and detection engineering.

Thought Process & Decision-Making
Splunk (SIEM), Suricata (IDS), pfSense (segmentation), and Nessus (vulnerability scans) were combined to simulate a functional SOC pipeline. Kali Linux was used for red team operations, with defensive actions completing the loop.

Technical Highlights

  • Centralized logs via Splunk with Sysmon and Suricata feeds.
  • Ran Nmap scans and basic exploits via Kali Linux.
  • Validated Suricata alerts against Splunk correlation rules.
  • Used Nessus to identify and patch known vulnerabilities.

What I Learned / What’s Next
The project emphasized thinking in terms of attack paths and detection gaps. Future plans involve extending the lab by integrating SOAR playbooks or building out MITRE-based detection rules in Splunk.
