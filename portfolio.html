Vulnerability Management & Compliance Framework Lab

Project Snapshot
Simulated real-world vulnerability testing to evaluate system compliance against industry-standard security frameworks like NIST, ISO 27001, and OWASP Top 10.

Context & Motivation
I wanted to explore how vulnerability scanning aligns with governance frameworks and control baselines. This lab helped me bridge technical execution with broader risk and compliance strategies.

Thought Process & Decision-Making
I selected systems with known weaknesses and scanned them using Nessus and OpenVAS. The focus was on detecting gaps against predefined controls. Mapping findings to NIST and ISO controls revealed inconsistencies in patching cadence and access controls. I prioritized fixes based on exploitability and regulatory relevance.

Technical Highlights

  • Used Tenable Nessus to detect critical vulnerabilities across Windows and Linux hosts.
  • Correlated findings with OWASP Top 10 and CIS benchmarks.
  • Created a remediation plan and measured improvements by re-running scans.
  • Logged all vulnerabilities and mitigations for audit reporting.

What I Learned / What’s Next
This lab deepened my understanding of how technical risk maps to compliance obligations. Next, I plan to automate baseline checks using scripts and integrate scan results with SIEM alerts.

AWS Cloud Security & Threat Detection Lab

Project Snapshot
Designed and deployed a cloud honeypot on AWS to collect real-world attack data and refine detection strategies using native AWS tools.

Context & Motivation
I wanted hands-on experience with cloud-native security controls and real-world telemetry. This lab gave me insight into attacker behavior and AWS threat visibility.

Thought Process & Decision-Making
T-Pot was deployed on an EC2 instance to simulate vulnerable services. I segmented traffic using VPCs and security groups to isolate malicious activity. AWS GuardDuty and Detective were used for detection and correlation. I tuned AWS WAF rules in response to observed brute-force attempts.

Technical Highlights

  • Deployed T-Pot honeypot stack on EC2.
  • Configured VPC Flow Logs, GuardDuty, and Detective for threat telemetry.
  • Captured and analyzed RDP/SSH brute-force attempts.
  • Created custom WAF rules to block malicious IPs in real-time.

What I Learned / What’s Next
This project taught me how AWS services interconnect for layered cloud defense. Next, I’ll explore Lambda automation for response and use Athena to query logs for long-term patterns.

SIEM & IDS/IPS Cybersecurity Simulation Lab

Project Snapshot
Built a hybrid SOC-style environment to monitor real-time security events, simulate threats, and execute incident response.

Context & Motivation
To get closer to real-world SOC operations, I built a custom lab to practice event triage, log correlation, and detection engineering.

Thought Process & Decision-Making
I combined Splunk for SIEM, Suricata for IDS, pfSense for segmentation, and Nessus for vulnerability scans. Each tool served a purpose in simulating a functional SOC pipeline. I used Kali Linux for red team ops and closed the loop with defensive actions.

Technical Highlights

  • Centralized logs via Splunk with Sysmon and Suricata feeds.
  • Simulated Nmap scans and basic exploits via Kali Linux.
  • Validated Suricata alerts against Splunk correlation rules.
  • Used Nessus to identify and patch known vulnerabilities.

What I Learned / What’s Next
I learned to think in terms of attack paths and detection gaps. I’d like to extend this lab by integrating SOAR playbooks or building out MITRE-based detection rules in Splunk.

SIEM & Security Control Validation Lab

Project Snapshot
Focused on validating existing security controls through targeted attack simulations and reviewing log coverage and correlation effectiveness.

Context & Motivation
I wanted to assess not just what tools were in place, but how effective they really were. This lab emphasized validation over implementation.

Thought Process & Decision-Making
Using open-source SIEM tools, I replayed known attack patterns and traced detection events. I modified log retention policies and dashboards to improve coverage and reduce false positives.

Technical Highlights

  • Used test scripts and attack frameworks to simulate credential dumping, privilege escalation, and lateral movement.
  • Verified alert generation in SIEM dashboards.
  • Tuned rules to reduce noise and emphasize high-fidelity signals.
  • Documented which controls were effective and which failed silently.

What I Learned / What’s Next
This reinforced that visibility without validation is a trap. I aim to continue refining detection coverage using threat emulation tools like Atomic Red Team and build reporting scripts for regular validation.
